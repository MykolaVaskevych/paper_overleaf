{\color{red}
This chapter presents a prototype DQN implementation on Atari Breakout, developed during the first semester as a proof of concept. The goal was to gain hands-on experience with the full training pipeline --- environment setup, policy configuration, callbacks, evaluation, and checkpointing --- before the main experiments began. Unlike the experiments in Chapters~4--6, hyperparameters were manually tuned here rather than kept at defaults, and no multi-seed statistical evaluation was applied. The chapter shows how to set up and run a DQN agent on an Atari game using Stable-Baselines3, explains the key configuration choices, and demonstrates that the pipeline produces a learning agent. The DQN algorithm itself is described in Chapter~\ref{chap:lit-review}.
}

\section{Implementation}

{\color{red}
Training a DQN agent on Atari with Stable-Baselines3 \citep{raffin2021sb3} requires three components: an environment stack, a model, and callbacks.

\textbf{Environment stack.}
Atari environments are created with \texttt{make\_atari\_env}, which applies the standard DQN preprocessing pipeline: grayscale conversion, $84\times84$ resizing, reward clipping, episodic life, and fire-on-reset. Eight parallel instances run simultaneously to improve data throughput. \texttt{VecFrameStack} concatenates four consecutive frames into a single observation so the agent can perceive motion --- a ball moving across the screen looks identical in a single frame, but the direction is recoverable from the stack. A separate single-environment setup is used exclusively for evaluation, ensuring clean measurement unaffected by training noise.

\textbf{Model.}
The \texttt{CnnPolicy} processes the stacked frames through a convolutional network (three conv layers followed by fully connected layers), identical to the architecture from \citet{mnih2015human}. Key hyperparameters used here: replay buffer of 400K frames, $\varepsilon$ annealed from 1.0 to 0.01, target network updated every 10,000 steps, batch size 64, learning rate $10^{-4}$. These were selected to approximate the original paper's settings. In contrast, the main experiments in Chapters~4--6 use a minimal model instantiation with no hyperparameter specification, relying entirely on SB3 defaults:
\begin{lstlisting}[language=Python, basicstyle=\small\ttfamily, frame=none]
model = DQN("CnnPolicy", vec_env)
\end{lstlisting}

\textbf{Callbacks.}
A \texttt{CheckpointCallback} saves the model and replay buffer every 500K steps, allowing training to be resumed. An \texttt{EvalCallback} runs the frozen policy on the evaluation environment every 100K steps, recording the mean return and saving the best model seen so far.

The complete training script is shown below.
}

\begin{lstlisting}[
    language=Python,
    numbers=left,
    numberstyle=\tiny,
    stepnumber=1,
    numbersep=5pt,
    breaklines=true,
    frame=shadowbox,
    basicstyle=\tiny\ttfamily,
    keywordstyle=\color{c++purple}\bfseries,
    stringstyle=\color{c++red},
    commentstyle=\color{c++green}
]
import ale_py
from stable_baselines3 import DQN
from stable_baselines3.common.env_util import make_atari_env
from stable_baselines3.common.vec_env import VecFrameStack
from stable_baselines3.common.callbacks import CheckpointCallback, EvalCallback

def main():
    vec_env = VecFrameStack(make_atari_env("BreakoutNoFrameskip-v4", n_envs=8, seed=42), n_stack=4)
    eval_env = VecFrameStack(make_atari_env("BreakoutNoFrameskip-v4", n_envs=1, seed=123), n_stack=4)

    checkpoint_callback = CheckpointCallback(save_freq=500_000, save_path="./checkpoints/",
        name_prefix="dqn_breakout", save_replay_buffer=True)
    eval_callback = EvalCallback(eval_env, best_model_save_path="./best_model/",
        log_path="./logs/", eval_freq=100_000, deterministic=True, render=False)

    model = DQN("CnnPolicy", vec_env,
        learning_rate=1e-4, buffer_size=400_000, learning_starts=50_000,
        batch_size=64, gamma=0.99, train_freq=4, gradient_steps=1,
        target_update_interval=10_000, exploration_fraction=0.1,
        exploration_initial_eps=1.0, exploration_final_eps=0.01,
        verbose=1, device="cuda")

    model.learn(total_timesteps=10_000_000,
        callback=[checkpoint_callback, eval_callback], progress_bar=True)
    model.save("dqn_breakout_final")
    vec_env.close()
    eval_env.close()

if __name__ == "__main__":
    main()
\end{lstlisting}

\section{Results}

{\color{red}
Figure~\ref{training_curves.png} shows the training and evaluation reward over 10 million steps. Evaluation reward rises steadily from near zero and plateaus at a moderate level, confirming that the agent learns a non-trivial policy from raw pixels. The loss curve stabilises after initial oscillation, consistent with the replay buffer accumulating enough diverse experience for stable Q-value updates. Scores are modest relative to what the original DQN paper achieved with longer training and a larger replay buffer --- but the prototype was not optimised for peak performance; it was built to validate the pipeline.

Figure~\ref{fig:proto-game-states} illustrates the difference between the start of an episode and a state reached by the trained agent: multiple rows of bricks have been cleared, and the agent has discovered the side-tunnel strategy that DQN is known to learn on Breakout \citep{mnih2015human}. A video of the trained agent playing is available at: \url{https://youtube.com/shorts/hdiVVfBSX9c?feature=share}
}
